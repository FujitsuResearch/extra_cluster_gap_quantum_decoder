# Soft-Output Union-Find Decoder

This decoder supports these soft-output methods:

* Extra-cluster gap ([Kishi et al., 2026](https://arxiv.org/abs/2602.03336))
* Bounded cluster gap ([Kishi et al., 2026](https://arxiv.org/abs/2602.03336))
* Cluster gap ([[Meister et al., 2024]](https://arxiv.org/abs/2405.07433))

## Note

This implementation likely works correctly only when logical operators are defined at the boundary of the surface code (e.g., circuits generated via `stim.Circuit.generated` satisfy this automatically).

## Installation

**No Rust installation is required.** A pre-compiled Linux binary is bundled inside the `.whl` file.

Download the `.whl` file from `/target/wheels` and install it as follows (a `venv` environment is recommended):

```bash
pip install additional_grown_decoder-2.3.2-cp310-abi3-manylinux_2_34_x86_64.whl
```

To uninstall:

```bash
pip uninstall additional_grown_decoder
```

## Usage

```python
from additional_grown_decoder import calc_signed_gap
```

Importing as above makes the following function available:

```python
def calc_signed_gap(
        circuit: stim.Circuit,
        num_shots: int,
        *,
        use_preskill: bool = False,
        use_bounded_dijkstra: bool = False,
        use_additional_growth: bool = False,
        additional_max_growth: float | None = None,
        additional_max_growth_and_returns_are_dB: bool = False,
        debug: bool = False,
        ) -> (
                list[list[bool]],
                list[list[bool]],
                dict[str, list[list[float]]],
                )
```

To use [Meister et al., 2024], pass `use_preskill=True`.  
To use extra-cluster growth, pass `use_additional_growth=True` and specify the maximum growth, e.g. `additional_max_growth=10`.

By default, `additional_max_growth` is in units of the decoding graph edge weights. Passing `additional_max_growth_and_returns_are_dB=True` switches to dB units, in which case the soft-output values in the return value are also in dB.

The first two return values, `detection_events` and `observables`, are identical to those from `sampler.sample(num_shots, separate_observables=True)`.  
The dictionary contains the following keys:

```python
{
    'sign': [...],
    # Sign of the decoding outcome per shot: -1 if a logical error (decoding failure) occurred, +1 otherwise.

    'preskill_signed_gaps': [...],
    # Soft-output value from [Meister et al., 2024] multiplied by the sign, per shot.
    # In dB if `additional_max_growth_and_returns_are_dB=True`.

    'preskill_num_nodes_of_decode_graph': [...],
    # Number of nodes in the quotient graph produced by [Meister et al., 2024], per shot.

    'preskill_actually_visited_nodes': [...],
    # Number of nodes actually visited by Dijkstra's algorithm in [Meister et al., 2024], per shot.

    'bounded_dijkstra_signed_gaps': [...],
    # Soft-output value from [Meister et al., 2024] multiplied by the sign, per shot,
    # computed with the search range limited to `additional_max_growth` for faster execution.
    # In dB if `additional_max_growth_and_returns_are_dB=True`.

    'bounded_dijkstra_actually_visited_nodes': [...],
    # Number of nodes actually visited by Dijkstra's algorithm in the bounded cluster gap, per shot.

    'additional_growth_simple_signed_gaps': [...],
    # Extra-cluster growth amount used directly as the soft output, multiplied by the sign, per shot.
    # `[None]` if the two boundaries were not connected within the maximum growth.
    # In dB if `additional_max_growth_and_returns_are_dB=True`.

    'additional_growth_cluster_graph_signed_gaps': [...],
    # Cluster-graph-based soft output from extra-cluster growth, multiplied by the sign, per shot.
    # `[None]` if the two boundaries were not connected within the maximum growth.
    # In dB if `additional_max_growth_and_returns_are_dB=True`.

    'additional_growth_num_nodes_of_cluster_graph': [...],
    # Number of nodes in the cluster graph for extra-cluster growth, per shot.
    # `[0]` if the two boundaries were not connected within the maximum growth.

    'additional_growth_actually_visited_nodes': [...],
    # Number of nodes actually visited in the cluster graph generated by extra-cluster gap with cluter graph.

    'grown_weight_ufd': [...],
    # Growth amount used by the UF decoder alone (excluding any extra-cluster growth), per shot.

    'debug_det_to_de': [...],
    # Mapping from each detector to the root of its cluster in the UF decoder alone, per shot,
    # The total number of detectors across all clusters equals exactly the number of
    # vertices used by the UF decoder.
    # Only present when `debug=True`.

    'debug_additional_det': [...],
    # List of detectors added to clusters during extra-cluster growth, per shot.
    # Only present when `debug=True`.
}
```
